# G4F - единая документация

Это единственный актуальный документ проекта.
Любая старая документация считается невалидной и не используется.

## 1. Состав репозитория

- `vscode/` - основная кодовая база (fork VS Code + интеграции Void/Qwen).
- `vscode/engine/void-scene-loader/` - загрузчик сцен `.vecn` для Bevy.
- `test world/` - тестовый Bevy-проект для проверки рендера и узлов сцены.
- `qwen-code-clean/`, `qwen-code-new/` - форки/варианты Qwen CLI и сопутствующая интеграция.

## 2. Текущий стандарт сцены

- Формат сцены: `.vecn`
- Базовый файл теста: `test world/assets/scenes/main.vecn`
- Основной загрузчик: `vscode/engine/void-scene-loader/src/lib.rs`

Что сейчас реализовано в рантайме:
- загрузка `WorldEnvironment` и базовых ресурсов сцены;
- procedural sky fallback (градиент + солнце + облака);
- fallback-визуализация для большинства узлов-стабов;
- расширенные runtime-логи по наличию/работе узлов;
- FPS debug player в `test world`.

## 3. Управление в test world

- `W` `A` `S` `D` - движение
- `Shift` - бег
- `Space` - прыжок
- `Ctrl` - присед
- ЛКМ по окну - захват курсора (mouse look)
- `Esc` - отпустить курсор

## 4. Запуск

```powershell
cd "C:\Users\Starred\Desktop\G4F\test world"
cargo run
```

## 5. Чеклист проверки

- В логах есть:
  - `Scene loaded`
  - `Scene spawned successfully`
  - `Skybox: ... (procedural clouds)`
- В блоке `Runtime Render Checks`:
  - `camera true work`
  - `mesh true work`
  - `directionallight true work`
- Игрок управляется с клавиатуры и мыши.

## 6. Правила по документации

- В проекте должен существовать только этот файл: `DOCS.md`.
- Новые `README.md`, `*.md`, `docs/` не добавлять.
- При изменении архитектуры/пайплайна обновляется только `DOCS.md`.

## 7. Правила разработки

- Исправлять причину в плагине/движке, а не маскировать проблему правкой одной сцены.
- Для новых узлов сначала:
  - корректный парсинг;
  - стабильный spawn;
  - лог статуса `true work` или понятный `true visual`.
- Любое временное решение должно быть явно заменяемым и не ломать runtime.

## 8. Ближайшие задачи (актуальный backlog)

- Довести узлы со статусом `true visual` до полноценной функциональности `true work`.
- Улучшить procedural sky до более физически правдоподобной модели (экспозиция/рассеяние).
- Унифицировать отображение узлов между viewport/editor/runtime.
