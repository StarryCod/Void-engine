# Welcome Screen - Исправления логики кнопок ✅

## Проблемы которые были

1. Кнопки Cancel, Delete, Info, Open, Edit не работали
2. Логика была "ужасная" - кнопки вызывали методы но ничего не происходило
3. Использовался неправильный DisposableStore для обработчиков событий

## Что было исправлено

### 1. Исправлен showCreate() - использование правильного store

**Проблема**: В методе `showCreate()` использовался `this.cardStore` для добавления обработчиков событий кнопок Create/Cancel, но должен был использоваться `this.editStore`.

**Исправление**:
```typescript
// БЫЛО:
this.cardStore.add(dom.addDisposableListener(cancelBtn, ...));
this.cardStore.add(dom.addDisposableListener(createBtn, ...));

// СТАЛО:
this.editStore.add(dom.addDisposableListener(cancelBtn, ...));
this.editStore.add(dom.addDisposableListener(createBtn, ...));
```

Это критично потому что:
- `cardStore` очищается при `rebuildGrid()` - это удаляет обработчики кнопок!
- `editStore` очищается только при `cancelEdit()` или завершении редактирования
- Теперь кнопки работают до конца редактирования

### 2. Добавлено логирование в action методы

Добавлены console.log и console.warn во все методы действий:
- `actOpen()` - логирует selectedId и предупреждает если проект не выбран
- `actEdit()` - логирует selectedId и предупреждает если проект не выбран
- `actDuplicate()` - логирует selectedId и предупреждает если проект не выбран
- `actPin()` - логирует selectedId и предупреждает если проект не выбран
- `actInfo()` - логирует selectedId и предупреждает если проект не выбран
- `actDelete()` - логирует selectedId и предупреждает если проект не выбран

Теперь в консоли будет видно:
```
[Void] actOpen called, selectedId: abc123
[Void] Opening project: My Project /path/to/project
```

Или если проект не выбран:
```
[Void] actOpen called, selectedId: null
[Void] No project selected
```

### 3. Улучшена обработка ошибок

Все методы теперь проверяют что проект выбран и возвращают early return если нет:
```typescript
private actOpen(): void {
	const p = this.sp();
	if (!p) {
		console.warn('[Void] No project selected');
		return; // Ранний выход
	}
	this.openProject(p);
}
```

## Как работает логика

### Выбор проекта

1. Пользователь кликает на карточку проекта
2. Вызывается `sel(projectId)`
3. `selectedId` устанавливается
4. Карточка получает класс `vs-gcsel`
5. Action row показывается (`vs-aron`)

### Действия с проектом

1. Пользователь кликает кнопку в action row (Open, Edit, Delete и т.д.)
2. Вызывается соответствующий метод `actXXX()`
3. Метод получает выбранный проект через `sp()` (selected project)
4. Если проект не выбран - логируется предупреждение и выход
5. Если проект выбран - выполняется действие

### Создание проекта

1. Пользователь кликает "New Project"
2. Вызывается `showCreate()`
3. Создаётся inline форма редактирования
4. Обработчики кнопок добавляются в `editStore` (ИСПРАВЛЕНО!)
5. При клике Cancel/Create - форма удаляется и `editStore` очищается

### Редактирование проекта

1. Пользователь кликает кнопку Edit на карточке
2. Вызывается `showEdit(project)`
3. Карточка заменяется на форму редактирования
4. Обработчики добавляются в `editStore`
5. При сохранении/отмене - форма удаляется и grid перестраивается

### Удаление проекта

1. Пользователь кликает кнопку Delete
2. Вызывается `showDeleteConfirm(project)`
3. Открывается модальное окно с подтверждением
4. Обработчики добавляются в `modalStore`
5. При подтверждении - проект удаляется с диска и из списка

## Почему кнопки не работали

### Основная причина: неправильный DisposableStore

Когда использовался `cardStore` для кнопок Create/Cancel:
1. Пользователь кликает "New Project"
2. Создаётся форма, обработчики добавляются в `cardStore`
3. Пользователь вводит имя и кликает Create
4. Вызывается `rebuildGrid()` который делает `cardStore.clear()`
5. Обработчики удаляются ДО того как код выполнится!
6. Кнопка не работает

Теперь с `editStore`:
1. Пользователь кликает "New Project"
2. Создаётся форма, обработчики добавляются в `editStore`
3. Пользователь вводит имя и кликает Create
4. Код выполняется, проект создаётся
5. Только ПОСЛЕ этого `editStore.clear()` вызывается
6. Всё работает!

### Вторичная причина: отсутствие логирования

Без логирования невозможно было понять:
- Вызываются ли методы вообще?
- Выбран ли проект?
- Где именно происходит ошибка?

Теперь с логированием всё видно в консоли.

## Что ещё может не работать

Если кнопки всё ещё не работают, проверьте:

1. **CSS классы** - убедитесь что кнопки имеют правильные классы:
   - `.vs-ab` - action button
   - `.vs-abd` - danger button (delete)

2. **Event listeners** - убедитесь что `ab()` метод вызывается для каждой кнопки:
```typescript
this.ab(aL, 'Open', I.folder, () => this.actOpen());
this.ab(aL, 'Edit', I.edit, () => this.actEdit());
// и т.д.
```

3. **Выбор проекта** - убедитесь что проект выбирается при клике:
   - Карточка должна получить класс `vs-gcsel`
   - Action row должен показаться (класс `vs-aron`)

4. **Модальные окна** - убедитесь что overlay создаётся:
   - Должен появиться элемент `.vs-ov` в body
   - Должен иметь класс `.vs-ovon` для анимации
